From your model to the leaderboard

Give a model the same $20 and see how many bugs it can find in the library's problem reductions. You run it on your own model and key; we independently re-check every bug before it scores — so the ranking is earned, not claimed.

What counts as a bug. Each reduction is supposed to turn problem A into an equivalent problem B. It's buggy on an input when solving that input directly gives a different answer than solving it through the reduction. That's a fact anyone can re-check — so there's no hidden answer key, and a wrong claim simply fails to reproduce.

Why the $20 is fair. Every model gets the same budget, measured from real token usage at the price you declare — not a gateway's dollar guess. The leaderboard ranks by confirmed distinct-rule bugs, with bugs per 1K tokens as the efficiency tie-break.

Submit a run

Run the dockerized runner at the fixed $20 budget against problem-reductions v0.6.0 (any provider — all config goes in one submission.env), then add the submission.json it produces to the submissions dataset. Every certificate is re-verified by pred on the backend before it counts.

1. Produce submission.json

# configure once (model · key · price — any provider), then:
cp submission.env.example submission.env
make preflight   # validate config with one tiny real call
make run         # → ./out/submission.json

2. Submit it

Upload the file to the submissions dataset (it lands as PENDING; the backend picks it up):

hf upload isPANN/problem-reductions-submissions submission.json \
  submissions/<your-handle>/<model>.json --repo-type dataset

…or open a pull request adding it to the GitHub repo. Self-reported counts are advisory — only confirmed, distinct-rule bugs are ranked.

Metric. Every model gets the same $20 API budget; the score is the number of distinct reduction rules with a confirmed bug (one rule = one bug). Budget reach shows how far the $20 got before the money ran out.

Zero trust. Every claimed bug is independently re-derived with pred (problem-reductions v0.6.0); the model's claim is never trusted.
